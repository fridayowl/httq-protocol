# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

## Reporting a Vulnerability

**DO NOT** create a public GitHub issue for security vulnerabilities.

Instead, please email: **security@httq.org**

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

We will respond within 48 hours and provide a timeline for fixes.

## Security Measures

HTTQ implements multiple security layers:

1. **Post-Quantum Cryptography**: Resistant to quantum attacks
2. **Perfect Forward Secrecy**: Each session uses ephemeral keys
3. **Constant-Time Operations**: Resistant to timing attacks
4. **Memory Safety**: Secure memory handling
5. **Input Validation**: All inputs sanitized

## Security Audits

We welcome security audits from:
- Security researchers
- Academic institutions
- Professional audit firms

Contact us at security@httq.org to coordinate.

## Responsible Disclosure

We follow responsible disclosure practices:
1. Report received
2. Vulnerability confirmed
3. Fix developed and tested
4. Security advisory published
5. Public disclosure (coordinated)
